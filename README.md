### Архитектурная «коробка»

1. **Единый сервер на FastAPI**
    
    - **Бот (aiogram):** Отвечает только за интерфейс, прием сообщений от пользователя и первичную валидацию команд.
    
    - **Бэкенд (FastAPI):** Хранит базу данных, обрабатывает тяжелые вычисления, управляет бизнес-логикой.
    
    - **Общение:** Бот делает запросы к FastAPI, получает данные и показывает их пользователю.
        
2. **База данных — SQLite**
    
    - Файл рядом с проектом, доступ через «простую» библиотеку (вероятно, синхронный `sqlite3).
        
3. **Админ и подписчик**

    - Админ работает через **веб-оболочку** (рендеринг форм от FastAPI) — заполняет темы, статьи, сохраняет.
        
    - Подписчик взаимодействует**только через Telegram-бота**: получает статьи.
        

### Поток данных (упрощённо)

- Админ заходит на веб-интерфейс, заполняет формы, нажимает «сохранить».
    
- FastAPI обрабатывает данные, пишет в SQLite.
